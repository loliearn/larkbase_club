## 1. 产品概述

### 1.1 产品愿景

> 让每个企业都能快速找到合适的飞书多维表格专家，让每位专家都能高效展示专业能力并获得商业机会。

### 1.2 核心价值主张

**对于企业客户：**

| 核心卖点 | 说明 |
|----------|------|
| **认证行家** | 经过平台认证的行家具备真实项目经验，企业可以快速筛选出可信的专业人士，无需逐一验证 |
| **智能匹配** | 提交需求后，系统根据技能、预算、评分自动推荐最合适的行家，省去大量筛选和比较的时间 |

**对于行家（后续版本）：**

| 核心卖点 | 说明 |
|----------|------|
| **获客效率** | 平台流量分发，降低获客成本 |
| **品牌背书** | 官方认证标识，建立个人行业影响力 |

### 1.3 差异化核心：认证体系

认证体系是平台的核心差异化机制。

| 认证等级 | 要求 | 平台标识 |
|----------|------|----------|
| **初级行家** | 完成基础信息 + 2个案例 | 无特殊标识 |
| **认证行家** | 5个+成功案例 | 蓝色「认证行家」徽章 |
| **金牌行家** | 20+成功案例 + 无投诉 | 金色「金牌行家」徽章 + 首页优先推荐 |
| **战略行家** | 平台邀请 + 年度GMV≥100万 | 紫色「战略行家」徽章 |

> 注1：MVP 阶段仅展示认证等级标识，认证审核流程由后台人工处理，不在 MVP 功能范围内。
> 注2：认证行家/金牌行家的评分要求待 V2 评价系统上线后补充。

### 1.4 MVP 范围

**MVP 目标：** 验证"认证体系 + 智能匹配"模式能否解决供需双方痛点

**MVP 包含的 5 个页面：**

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| 首页 | `/` | 品牌展示、行家/模板入口 |
| 行家列表 | `/experts` | 行家筛选、搜索 |
| 行家详情 | `/experts/[id]` | 行家档案、案例、联系方式 |
| 模板列表 | `/templates` | 模板分类、筛选、搜索 |
| 提交需求 | `/requests/new` | 3步骤需求表单 |

**MVP 不包含的功能（V2/V3）：**

- 在线支付、合同签署
- 双向评价系统
- 在线沟通（站内信）
- 匹配结果页（实时展示在提交需求页完成后）
- 行家中心（行家入驻后的管理后台）

### 1.5 商业模式

**收入来源：** 交易佣金（按成交金额的 10-15% 收取）

**定价策略：** 行家自主定价，平台提供市场参考价，不干预定价

**MVP 阶段：** 暂不实现支付功能，需求匹配后由双方线下沟通成交

## 2. 目标用户与使用场景

### 2.1 目标用户

**行家（服务提供方）**

| 来源 | 特征 |
|------|------|
| 独立开发者 / 自由职业者 | 50%，灵活、专业能力强、获客困难 |
| 飞书官方认证服务商 | 30%，有官方背书、但缺乏展示平台 |
| 企业IT团队成员 | 15%，有企业背景、兼职接私活 |
| 飞书社区KOL / 讲师 | 5%，有影响力、但服务能力待验证 |

**企业客户（服务需求方 - B端）**

| 规模 | 占比 | 特征 |
|------|------|------|
| 小微企业（1-50人） | 40% | 需求简单、预算有限、决策快 |
| 中小企业（50-200人） | 45% | 需求中等、预算适中、决策链短 |
| 中大型企业（200-1000人） | 15% | 需求复杂、预算充足、决策周期长 |

**个人用户（服务需求方 - C端）**

飞书多维表格的个人爱好者、小团队负责人，通过模板购买或咨询解决问题。

### 2.2 典型使用场景

**场景 A：创业公司快速搭建 CRM**

> 小明是某电商创业公司的运营负责人，公司30人，想用飞书多维表格管理客户跟进。
>
> **用户路径：** 访问首页 → 搜索「CRM」或浏览行家列表 → 查看行家详情和案例 → 提交需求 → 获得推荐行家 → 线下联系成交

**场景 B：制造业企业数字化转型**

> 某机械设备制造企业IT负责人，150人，需要搭建生产进度管理和订单追踪系统。
>
> **用户路径：** 访问首页 → 筛选「生产制造」行业行家 → 查看多个行家档案和评分 → 提交需求（包含预算和时间要求）→ 获得匹配行家推荐 → 选择最合适的行家合作

**场景 C：行家获客**

> 张明是独立开发者，擅长飞书多维表格开发，通过朋友介绍客户为主，缺乏稳定获客渠道。
>
> **用户路径（当前 MVP 暂不包含行家入驻流程）：** 完成行家档案（后台人工审核）→ 档案展示在平台 → 企业客户浏览并提交需求 → 收到站内+邮件通知 → 线下沟通成交

**场景 D：个人用户购买模板**

> 小李是某小团队负责人，想找一个现成的客户管理模板。
>
> **用户路径：** 访问模板列表 → 筛选行业/分类 → 查看模板详情 → 联系模板作者（行家）→ 获取模板或定制服务

## 3. 功能详情

### 3.1 首页

**页面目的：** 品牌展示 + 流量分发入口，让用户快速找到目标页面。

**页面结构：**

| 区块 | 内容 | 跳转目标 |
|------|------|----------|
| Hero | 主标题 + 副标题 + 搜索框 + CTA按钮 | 搜索触发 → 行家列表；CTA → 提交需求 |
| 平台数据 | 行家数、案例数、匹配率等信任背书数字 | 静态展示 |
| 推荐行家 | 3-6 位认证行家卡片 | 行家详情页 |
| 热门模板 | 3-6 个模板卡片 | 模板详情页 |
| 最新博客 | 2-3 篇博客文章入口 | 博客详情页（V2） |
| 招募行家 CTA | 「成为行家」入口 | 行家入驻引导（V2） |

**搜索行为：**

- 用户在搜索框输入关键词（行家姓名、模板关键词）
- 搜索框回车 → 跳转行家列表页，并自动填入关键词过滤

**设计规范：**

- 最大内容宽度：1280px
- 响应式断点：Mobile < 640px / Tablet 640-1024px / Desktop > 1024px
- 主色调：飞书蓝 #005da7
- 无 1px 边框分割，使用背景色位移区分区块

### 3.2 行家列表页

**页面目的：** 让企业客户快速筛选和找到合适的行家。

**筛选维度：**

| 筛选字段 | 说明 | 数据来源 |
|----------|------|----------|
| 认证等级 | 全部 / 认证行家 / 金牌行家 / 战略行家 | 行家档案必填 |
| 技能标签 | 多选（多维表格、自动化、API集成、数据看板、CRM系统等） | 行家档案必填，最多10个 |
| 价格范围 | 预算区间（最小~最大） | 行家档案必填（时薪或起步价） |
| 所在城市 | 下拉选择 | 行家档案选填 |

**排序方式：**

| 排序 | 说明 |
|------|------|
| 默认（智能排序） | 综合认证等级、项目完成数、匹配度 |
| 项目数最多 | 按累计完成项目数降序 |
| 价格最低 / 最高 | 按时薪或起步价排序 |

**行家卡片信息：**

每个行家卡片展示：
- 头像
- 姓名
- 认证等级徽章（无 / 蓝色认证 / 金牌 / 战略）
- ⭐4.8（占位，MVP 硬编码，V2 评价系统上线后替换为真实评分）
- 擅长技能标签（最多显示4个）
- 城市（选填）
- 时薪或起步价
- 累计完成项目数

**分页：** 列表底部使用分页组件，每页显示 12-20 条。

**搜索：** 顶部搜索框支持按行家姓名搜索，实时过滤列表。

**空状态：** 无匹配结果时显示空状态提示，提供「清除筛选」操作。

### 3.5 提交需求页

**页面目的：** 收集企业客户（或个人用户）的需求信息，通过智能匹配算法推荐合适的行家。

**交互方式：** 3步骤表单，同页面内切换，无需跳转新页面。提交后，同页面切换至匹配结果展示。

---

**步骤 1：基本信息**

| 字段 | 类型 | 说明 |
|------|------|------|
| 项目名称 | 文本输入，必填 | 如「企业销售管理系统定制」 |
| 项目类型 | 单选，必填 | 模板定制 / 系统开发 |
| 预算范围 | 区间输入（最小~最大，元），必填 | 如 ¥5,000 ~ ¥20,000 |

---

**步骤 2：需求详情**

| 字段 | 类型 | 说明 |
|------|------|------|
| 所属行业 | 下拉选择，必填 | 零售/电商、服务/咨询、制造/贸易、科技/互联网、教育/培训、金融、医疗、其他 |
| 企业规模 | 单选，必填 | 1-50人 / 50-200人 / 200-500人 / 500-1000人 / 1000人以上 |
| 需求描述 | 多行文本，必填 | 详细描述需求，包括：当前痛点、期望实现的功能、特殊要求等 |

---

**步骤 3：联系方式**

| 字段 | 类型 | 说明 |
|------|------|------|
| 联系人姓名 | 文本，必填 | |
| 公司名称 | 文本，选填 | |
| 联系电话 | 文本，必填 | |
| 邮箱 | 文本，必填 | |

---

**智能匹配逻辑（中级版）：**

提交后实时计算，匹配权重：

| 维度 | 权重 | 计算规则 |
|------|------|----------|
| 技能匹配度 | 40% | 需求类型与行家服务类型完全匹配 = 100%，不匹配 = 0%（行家可提供模板定制/系统开发二种服务，需求类型为二选一，直接比对） |
| 价格匹配度 | 30% | 行家价格在预算范围内 = 100%，超出 <20% = 60%，超出 ≥20% = 0% |
| 项目经验 | 30% | ≥20个项目 = 100%，≥10个 = 80%，≥5个 = 60%，<5个 = 40% |

> 注：MVP 阶段「评分」无真实数据，评分维度暂替换为「项目完成数」。后续 V2 评价系统上线后，替换回评分。

推荐阈值：≥70 分进入推荐列表
推荐数量：按得分降序，最多展示 5 位行家

---

**匹配结果展示（同页面切换）：**

提交成功后，页面切换至匹配结果区域，展示：

- 成功提示（需求提交成功）
- 匹配行家列表（按匹配得分降序）
  - 每位行家展示：头像、姓名、认证等级徽章、擅长技能、价格、匹配得分、匹配原因标签、项目完成数
  - 「查看详情」按钮 → 跳转行家详情页
  - 「获取联系方式」按钮 → 点击展开联系方式（微信/邮箱）；再次点击收起

**无匹配结果：** 若无行家达到推荐阈值，显示「暂无匹配行家，我们将尽快处理您的需求」提示。

## 4. 核心用户交互流程

### 4.1 企业客户流程

```
发现 ────────→ 评估 ────────→ 提交需求 ────────→ 获得匹配
  │              │                │                │
  ▼              ▼                ▼                ▼
 访问首页      浏览行家         填写表单         查看推荐
 搜索需求      查看档案         3步骤提交        获取联系方式
 朋友推荐      查看案例         实时匹配        线下沟通
               了解价格
```

**详细用户路径：**

1. **发现/认知**
   - 用户通过搜索引擎、飞书官方渠道、朋友推荐等方式了解到平台
   - 访问首页，了解平台定位和核心价值

2. **评估/筛选**
   - 访问行家列表页，通过认证等级、技能标签、价格、城市等维度筛选
   - 点击行家卡片进入行家详情页，查看案例、简介、价格
   - 可浏览模板列表寻找灵感

3. **提交需求**
   - 进入提交需求页，完成3步骤表单
   - 系统实时计算，页面内切换展示匹配结果

4. **获得匹配**
   - 查看推荐行家列表，获取联系方式
   - 与行家线下沟通，确定合作

### 4.2 行家流程

```
入驻 ────────→ 完善档案 ────────→ 接收需求 ────────→ 完成服务
  │               │                │                │
  ▼               ▼                ▼                ▼
 申请入驻      上传案例         收到通知        线下沟通
 人工审核      设置价格         需求详情        项目交付
 获得初级认证   发布模板         沟通报价        （后续版本：收款）
```

**详细用户路径：**

1. **入驻申请（V2）**
   - 行家通过行家入驻入口提交申请（邮箱注册）
   - 平台后台人工审核，审核通过后获得「初级行家」资格

2. **完善档案**
   - 登录后进入行家中心（V2）
   - 编辑个人信息：姓名、头像、头衔、简介
   - 编辑专业信息：技能标签、城市、从业年限
   - 设置服务类型和价格
   - 上传成功案例（至少2个）

3. **接收需求**
   - 收到站内通知 + 邮件通知（有新需求匹配）
   - 登录行家中心查看需求详情
   - 线下与需求方沟通

4. **完成服务**
   - 项目交付
   - （V2）双方在平台确认完成，平台收取佣金

### 4.3 产品信任体系说明

**信任是平台的核心差异化机制。**

| 信任机制 | 说明 | MVP 实现 |
|----------|------|----------|
| 认证等级体系 | 初级→认证→金牌→战略，区分行家专业能力 | 仅展示，审核人工处理 |
| 成功案例展示 | 行家上传真实项目案例，证明实操能力 | 展示，无审核流程 |
| 价格透明 | 行家明码标价，企业有参考依据 | 展示，无管控 |
| 评分评价（V2） | 双向评分，帮助后来者决策 | MVP 暂不实现 |

> **MVP 信任机制局限性：** MVP 阶段认证审核为人工处理，无自动审核机制；无评价系统，无法通过历史评分判断行家质量。需要在 V2 版本中完善。

### 5.2 通用组件

**按钮组件：**

| 类型 | 样式 | 用途 |
|------|------|------|
| Primary | 蓝色实底 #005da7，白色文字 | 主要操作（提交、获取联系方式） |
| Secondary | 无底，灰色描边 | 次要操作（取消、返回） |
| Ghost | 无底无描边 | 辅助操作（查看更多） |

按钮圆角：8px
点击反馈：scale(0.98)

**卡片组件：**

- 圆角：12px
- 背景：白色（#ffffff）
- 无描边，用背景色位移或间距区分
- 悬停：轻微上浮 + 阴影加深

**徽章组件：**

| 等级 | 样式 |
|------|------|
| 初级行家 | 无徽章 |
| 认证行家 | 蓝色背景 #E8F2FC + 蓝色文字「认证行家」|
| 金牌行家 | 金色背景 #FEF3CD + 金色文字「金牌行家」|
| 战略行家 | 紫色背景 #F3E8FC + 紫色文字「战略行家」|

**表单输入框：**

- 无背景填充
- 底部描边（#c1c7d3，40%透明度）
- 聚焦：底部变 2px 蓝色 #005da7

**步骤指示器（提交需求页）：**

- 圆形数字步骤 + 文字标签
- 已完成：蓝色实底 + ✓
- 当前：蓝色描边 + 蓝色数字
- 待完成：灰色描边 + 灰色数字
- 步骤间用横线连接，已完成部分蓝色，未完成部分灰色

## 6. 非功能需求

### 6.1 性能

| 指标 | 目标 |
|------|------|
| 首屏加载时间 | < 3 秒 |
| 页面切换响应 | < 1 秒 |
| 匹配计算 | < 2 秒（实时计算无 loading 状态） |

### 6.2 兼容性

- Chrome / Edge / Safari / Firefox 最新两个版本
- 移动端：iOS Safari 14+ / Android Chrome 100+
- 响应式支持：Mobile / Tablet / Desktop 三断点

### 6.3 SEO

- 首页、行家列表、行家详情页支持基础 SEO（标题、描述、Open Graph）
- 行家详情页使用 SSR，确保搜索引擎可抓取
- 未来博客页需主动提交 sitemap

### 6.4 数据安全

- 用户联系方式仅在提交需求后展示给匹配行家
- 表单数据不做持久化存储（MVP 阶段，匹配结果仅展示，不存储）
- 敏感操作（登录、提交需求）使用 HTTPS

### 6.5 可维护性

- 使用 TypeScript 严格模式
- 组件库使用 Tailwind CSS + 自定义设计系统
- Next.js App Router，页面级服务端渲染
- 目录结构清晰，组件按 feature 组织
